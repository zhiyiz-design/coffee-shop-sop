# 咖啡店 SOP 微信小程序

这是咖啡店 SOP 的微信小程序版本，适合员工直接从微信里打开使用。

## 功能

- 查看全部饮品 SOP
- 搜索饮品、原料、步骤
- 展开/收起卡片
- 编辑饮品名称、价格、原料、步骤、注意事项
- 新增/删除饮品
- 导出/导入 JSON 备份
- 通过微信云开发同步到所有员工手机

## 目录

- `app.json` 小程序配置
- `pages/index/` 首页
- `data/default-drinks.js` 默认菜谱
- `cloudfunctions/sopApi/` 云函数，负责读取和保存云端 SOP

## 第一次打开

1. 下载并打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择：`/Users/zoeymac/Documents/Work/coffe/wechat-miniprogram`
4. AppID 填你的小程序 AppID。
5. 后端服务选择“微信云开发”。
6. 导入后，进入“云开发”，创建一个云环境。

如果微信开发者工具没有自动识别 AppID，把 `project.config.json` 里的 `appid` 从 `touristappid` 改成真实 AppID。

## 配置云开发

1. 在微信开发者工具顶部点击“云开发”。
2. 创建云环境，记下环境 ID。
3. 如果工具没有自动选择环境，打开 `app.js`，把 `CLOUD_ENV_ID` 改成你的环境 ID。
4. 在云开发数据库里创建集合：`coffee_sop_drinks`。
5. 右键 `cloudfunctions/sopApi`。
6. 选择“上传并部署：云端安装依赖”。
7. 回到模拟器，点击“刷新”。

第一次刷新时，如果云端集合为空，云函数会自动写入默认 10 款 SOP。

## 员工使用

- 平时查看：直接打开小程序。
- 修改 SOP：点右上角“编辑”，改完点“保存云端”。
- 其他员工：点“刷新”就会看到最新版本。

当前策略是“最后一次保存生效”，适合你们现在不同时编辑同一条 SOP 的工作方式。

## 预览和发布

1. 在微信开发者工具点“预览”，用手机扫码测试。
2. 没问题后点“上传”。
3. 到微信公众平台提交审核。
4. 审核通过后发布。

## 还保留网页版本吗？

建议保留。小程序作为主入口，网页作为备用入口。两边目前不是同一个数据库：网页使用 Firebase Firestore，小程序使用微信云开发数据库。
